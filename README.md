# Резюме-сайт Євгенія Кофана

## Редагування тексту

Увесь текст сайту винесено в окремі мовні файли:

- `locales/uk.json` — українська версія, основна мова;
- `locales/en.json` — англійська версія.

Для зміни тексту відредагуйте значення праворуч від відповідного ключа. Назви ключів і CSS-селектори змінювати не потрібно.

## Локальний перегляд

Через використання JSON-файлів сайт потрібно відкривати через локальний сервер, а не подвійним кліком по `index.html`.

Наприклад:

```bash
python3 -m http.server 8000
```

Після цього відкрийте `http://localhost:8000`.

## GitHub Pages

1. Завантажте вміст цієї папки в корінь GitHub-репозиторію.
2. Відкрийте `Settings` → `Pages`.
3. Оберіть `Deploy from a branch`.
4. Вкажіть гілку `main` і папку `/root`.

## Візуальний редактор і публікація на GitHub

Відкрийте `editor.html` через опублікований GitHub Pages сайт, наприклад:

```text
https://USERNAME.github.io/REPOSITORY/editor.html
```

Редактор дозволяє завантажити `locales/uk.json` і `locales/en.json`, змінити текст та опублікувати оновлення безпосередньо в GitHub.

### Налаштування токена

Створіть **fine-grained personal access token** для конкретного репозиторію та надайте йому мінімальний дозвіл:

```text
Repository permissions → Contents → Read and write
```

У редакторі введіть:

- token;
- власника репозиторію;
- назву репозиторію;
- гілку, зазвичай `main`;
- шлях до сайту, якщо він лежить не в корені репозиторію.

Токен зберігається тільки в `sessionStorage` поточної вкладки. Не додавайте токен у файли сайту, GitHub-репозиторій або скриншоти.

Після натискання **«Опублікувати на GitHub»** редактор оновить лише змінені мовні JSON-файли. GitHub Pages застосує зміни після завершення deployment.

## Якщо публікація не працює

Редактор показує точну відповідь GitHub API. Перевірте:

- fine-grained token має доступ саме до цього репозиторію;
- Repository permissions → Contents встановлено в Read and write;
- правильно вказані власник, назва репозиторію, гілка та шлях до сайту;
- якщо сайт лежить у корені репозиторію, поле шляху має бути порожнім;
- якщо сайт лежить у папці docs, укажіть docs;
- захищена гілка може забороняти прямі коміти.

Після локального завантаження JSON зміни тепер також можна публікувати напряму: редактор перед записом сам отримує актуальний SHA файлу з GitHub.
